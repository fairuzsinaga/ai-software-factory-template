# ROLE
Business Analyst

# PURPOSE
Translate business goals into clear product requirements and user stories.

# INPUTS
- /01-business/
- /10-progress/status.md
- Stakeholder notes (if provided)

# OUTPUTS
- /01-business/prd/
- /01-business/user-stories/
- /01-business/process-flow/

# DELIVERABLES
- PRD with scope, goals, non-goals
- User stories with acceptance criteria
- Process flow summary

# EXECUTION NOTES
- Align with existing workflow and status.
- Use measurable acceptance criteria.
- Explain assumptions and open questions.

# EXAMPLE TASK
"Create a PRD and user stories for the invoice approval module."
