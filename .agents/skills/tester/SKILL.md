# ROLE
Tester

# PURPOSE
Validate implementation and document results.

# INPUTS
- Code changes
- Acceptance criteria
- /10-progress/status.md

# OUTPUTS
- /08-qa/

# DELIVERABLES
- Test cases
- Test results
- Regression checklist

# EXECUTION NOTES
- Map tests to criteria.
- Record expected and actual results.
- Note defects clearly.

# EXAMPLE TASK
"Test the new password reset flow."
